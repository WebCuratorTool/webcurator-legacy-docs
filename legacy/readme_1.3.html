Feature Enhancements

Added 'Member of' tab to Group display

This enables users to determine whether a Group is a Sub-Group of another.

Added a new Reports menu page

The Reports menu page displays the report name and description along with a link to the report.

Added the ability to edit and delete Annotations (system wide)

This has been applied to all screens that used annotations. It helps to prevent duplicate entry of annotations as users can now amend information when correcting mistakes.

Added indication of required fields (system wide)

Implemented by marking required fields with a red asterix '*', this amendment makes WCT more user friendly.

Automatic update of Permission state

When WCT is used to print or email a harvest authorisation request to an agent, the status is changed from 'pending' to 'requested'.

Added the ability to delete queued Target Instances

Users are able to delete a queued Target Instance straight away rather than waiting for it to become scheduled or running before deleting it. This allows other queued or scheduled Target Instances to move forward in the queue quicker.

Added a 'view' icon to the Users, Roles, and Agencies screens

Users can view records without being able to edit them.

Added a 'view' icon to the 'Action' column of the Permission Request Template list

Users can view records without being able to edit them.

Added an 'Access' tab to the Target and Group screens

This provides a place to store access related metadata which may be utilised by a web archive access tool.

Added a 'Display' tab to the Target Instance screen

This provides a place to store access related metadata which may be utilised by a web archive access tool.

Allow users to select number of results per page

Added a drop-down menu to the bottom of the search forms for setting the number of results per page (system wide). Possible values are 10, 20, 50, 100. The system remembers the user's preference using a cookie.

Modified Harvest Authorisations search results page

Added a new column before the action column labelled 'Status'. This displays the status of the permission record (or records) for this item.

Modified the Harvest Authorisation search form fields

Added a drop down menu to the Harvest Authorisation search form to filter Harvest Authorisations by Agency. Added a textbox to the search form called 'Permissions File Reference' for searching the Permissions 'File Reference' field (stored in a permissions record on the Permission tab). Added a textbox to the search form for searching 'URL Pattern' fields. Added checkboxes to the search form for searching on the Permissions 'Status' field.
Known Issue:  There is a bug in the hibernate libraries which causes a slight anomaly in the page/item numbering on results pages where duplicate rows are not shown. This bug does not affect the overall functionality.

Added the facility to filter Harvester Profiles, Users & Roles by Agency

Added a drop down list box (similar to the Targets screen) so lists on the Profiles, Users and Roles screens can be filtered by agency, with the lists filtered by the users' agency by default.

Modified Groups search form fields to filter by Group Type

Added a drop down list to the Groups Search Form to filter Groups by Type. Added a new column to the results page called 'Type' which will display the Group Type.

Changed the default search when opening the Groups screen

Changed the default search to display results filtered by Agency only.

Added the facility to 'Unclaim' Tasks from within the In-Tray

Added a new icon similar to the claim icon (with alt text 'Unclaim') which appears as an action once a user claims a task. When clicked the task is released - meaning the task will reappear in the In Trays of all the users with the appropriate privileges.

Made 'Target Reference Number is Mandatory' optional - according to system configuration

Added a property section to the relevant system configuration file called ‘TargetReferenceIsMandatory’, defaulting to true. If true the behaviour doesn’t change, if false, the Target Reference Number is no longer mandatory, and no error will occur without one. This allows a quicker workflow for the British Library without causing any problems for NLNZ.

Automated Modification Metadata in Prune Tool

The WCT now explicityly records whetehr a Harvet Result is the original harvest or has been modified. The Prune tool will record all items pruned from the harvest result, and this information will be reported in the SIP.

Harvest History Quality Review Tool

A new Quality Review tool has been created that will allow users to check how the current harvest compares with previous harvests. This can be useful to compare that the number of URLs harvests is similar to the number of URLs harvested in the past. A major reduction in the number of URLs or the size of the harvest may indicate that the harvest was not successful.

Paged Search Results

The search results screens now display a count of the number of results as well as the current page number and the number of pages in the result set. It is now possible to jump to any page in the result set.

Browse Tool Updates

In the Quality Review Tools menu, new links have been added to link to the live website and to the archived version. By default the archived version links to the International Web Archive, though this can be customised to point to your repository if desired.

Prune Tool Updates

The purne tool has been updated to provide more information about the website including:

  • The HTTP status code
  • The size of the resource
  • The size of the resource
  • The number of child URLs
  • The number of successfully downloaded child URLs
  • The number of failed downloaded child URLs
  • The size of the resource and its children

It is now also possible to choose a URL and jump into the browse tool.

Targets and Groups Searchable by Group

It is now possible to search Targets and Groups based on the groups to which they belong.

Crawler Activity Report

A new report has been created that will give an overview of the Harvest Agent's activities. This report includes information such as:

  • Target name
  • Status of harvest
  • The size of the resource
  • Start/end dates
  • Crawl duration
  • Bytes downloaded
  • Name of harvest agent

Notification and Task Controls

A user now has finer grained control over the delivery of their notification and tasks. They may choose to whether or not to receive certain notifications, and independently choose whether notifications or tasks are also sent to their email account.

Add Targets to Groups in Target Workflow

A new Groups tab has been added to the Target User Interface. This allows the user to select the parent groups for a given target. This is in addition to being able to also add targets to groups from the Groups user interface.

Log Viewer Enhancements

It is now possible to download the full log files from the WCT in addition to viewing them online. Downloading the full log file allows the user to view all information without being subject to the size restrictions placed on the online viewer.

Delete All Notifications

A new button has been added to the WCT Inbox that allows the user to delete all of their notifications.

Description Field Length Extended

The length of the description fields has been extended from 255 characters to 4000 characters.

Notification of Ownership Change

A notification is sent original owner of an object when ownership is transferred away from them.

Harvest Agent Field for Target Instance Objects

A target instance now holds the name of the Harvest Agent to which is was assigned. This is a read-only attribute, set by the system at the time the harvest is assigned. This information is included in the SIP.

Search ID Fields

It is now possible to search the main objects by their object identifier.

Target Instance Notification Messages

A number of target instnance notification messages have had minor changes to their format.

Interval Schedule Builder

A new mechanism has been created for setting schedules on targets and groups. The new mechanism allows a user to choose one of the following interval types:

  • Daily
  • Weekly
  • Monthly
  • Bi-monthly
  • Quarterly
  • Half yearly
  • Annually

These can then be configured with a time and other options. This new schedule builder is a simpler way to create regular schedules than using cron expressions in the custom schedule builder.

The custom schedule builder is still available and can be used when the new schedule builder does not offer enough flexibility.

Indicate when Permission Records are Expired or Future

The Target's seed list now shows a warning if one of the attached permissions is a future or expired record.

Harvest Now Button

The Harvest Now button is no longer on the Target Instance view screen. Instead, it has been moved directly on to the Target Instance search results screen, making it faster to launch a harvest.

Tab order and Names

A couple of the tabs have been re-ordered or renamed for consistency with the WCT home page. The reports tab is no longer a main-level tab and now resides uner the Management section.

Transfer Permission Button Moved

The Transfer Permissions button has been moved from the view permission screen to the edit permission screen.

Input Fields Resized

A number of input fields have been resized to display a larger number of characters on-screen.

MySQL Support

To support MySQL databases, the tablenames and fieldnames are now consistently in upper case.

The documentation includes instructions for deploying against MySQL.

Date Validation

All dates must now be entered with 4-digit years.

Internationalisation Updates (Library of Congress)

Notification messages and many JSP messages have been internationalised allowing libraries to localise messages for different languages or terminology. The focus of these changes has been to allow the terminology to be modified between different English speaking countries. While the underlying architecture will support language translation, not all messages have been extracted from the JSPs at this point.
